# 设备管理模块FAQ

## 常见问题

### 1. 设备信息显示不完整怎么办？

**可能原因**:
- 系统权限不足
- 设备驱动问题
- 信息采集超时

**解决方案**:
1. 检查应用运行权限
2. 更新设备驱动程序
3. 刷新设备信息
4. 查看系统日志获取详细错误

### 2. 监控数据不更新怎么办？

**可能原因**:
- 数据采集服务异常
- 网络连接问题
- 缓存未刷新

**解决方案**:
1. 检查数据采集服务状态
2. 验证网络连接
3. 手动刷新监控数据
4. 清除缓存后重试

### 3. 如何查看CPU温度？

**查看方法**:
1. 进入设备管理 > CPU详情
2. 查看温度传感器数据
3. 注意：需要系统支持温度传感器

**前提条件**:
- 主板支持温度传感器
- 系统已安装lm-sensors
- 已执行sensors-detect配置

### 4. 内存使用率一直很高怎么办？

**排查步骤**:
1. 查看进程内存占用排行
2. 检查是否有内存泄漏
3. 清理缓存和临时文件
4. 考虑增加物理内存

### 5. 磁盘I/O性能如何测试？

**测试方法**:
1. 进入设备管理 > 磁盘详情
2. 点击"性能测试"按钮
3. 选择测试类型（顺序/随机读写）
4. 查看测试结果报告

**注意事项**:
- 测试会影响磁盘性能
- 建议在业务低峰执行
- 避免对系统盘进行高强度测试

### 6. 网络设备显示异常怎么办？

**排查步骤**:
1. 检查网络接口状态
2. 验证网络配置
3. 重启网络服务
4. 检查物理连接

### 7. 如何设置设备告警阈值？

**设置方法**:
1. 进入设备管理页面
2. 点击"告警设置"
3. 选择监控指标
4. 设置阈值和通知方式
5. 保存配置

### 8. 设备诊断报告如何解读？

**报告内容**:
- 设备健康评分
- 性能瓶颈分析
- 潜在问题预警
- 优化建议列表

**建议操作**:
- 优先处理高风险问题
- 定期执行诊断
- 记录历史报告对比

### 9. 如何导出设备信息？

**导出方法**:
1. 进入设备列表页面
2. 点击"导出"按钮
3. 选择导出格式（JSON/CSV）
4. 选择导出范围
5. 下载导出文件

### 10. 支持监控哪些设备类型？

**支持列表**:
- CPU：型号、核心数、频率、温度
- 内存：容量、使用率、频率
- 磁盘：容量、I/O、健康状态
- 网络：接口、流量、连接数
- GPU：型号、显存、使用率（部分支持）

## 故障排查指南

### 设备信息问题
```
错误: 无法获取设备信息
排查:
1. 检查系统权限
2. 验证设备驱动
3. 查看系统日志
4. 重启信息采集服务
```

### 监控问题
```
错误: 监控数据异常
排查:
1. 检查数据采集进程
2. 验证数据源配置
3. 清除缓存数据
4. 重启监控服务
```

### 诊断问题
```
错误: 诊断执行失败
排查:
1. 检查诊断权限
2. 验证诊断工具安装
3. 查看诊断日志
4. 尝试简化诊断范围
```

## 最佳实践

### 监控建议
- 设置合理的告警阈值
- 定期检查监控趋势
- 关注异常波动
- 保留历史数据

### 诊断建议
- 定期执行设备诊断
- 关注健康评分变化
- 及时处理潜在问题
- 记录优化效果

### 运维建议
- 建立设备档案
- 定期更新设备信息
- 监控设备生命周期
- 制定升级计划
