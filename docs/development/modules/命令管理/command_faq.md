# 命令管理模块FAQ

## 常见问题

### 1. 如何执行命令？

**操作步骤**:
1. 进入命令管理页面
2. 在命令输入框输入命令
3. 点击"执行"按钮
4. 查看实时输出结果

### 2. 命令执行超时怎么办？

**解决方案**:
- 检查命令是否需要更长时间
- 考虑使用后台执行模式
- 将命令拆分为多个步骤
- 检查系统资源状态

### 3. 如何创建脚本模板？

**创建步骤**:
1. 进入脚本库页面
2. 点击"创建脚本"
3. 输入脚本名称和描述
4. 编写脚本内容
5. 设置脚本参数
6. 保存脚本

### 4. 脚本支持哪些参数类型？

**支持类型**:
- 字符串参数
- 数字参数
- 布尔参数
- 选择列表参数
- 文件路径参数

### 5. 如何查看历史命令？

**查看方法**:
1. 进入命令历史页面
2. 浏览历史记录列表
3. 使用搜索功能筛选
4. 点击查看详情

### 6. 历史记录可以保留多久？

**保留策略**:
- 默认保留1000条记录
- 可在设置中调整保留数量
- 支持手动清理历史

### 7. 如何重新执行历史命令？

**操作方法**:
1. 在历史列表找到目标命令
2. 点击"重新执行"按钮
3. 确认执行操作
4. 查看执行结果

### 8. 命令执行失败如何排查？

**排查步骤**:
1. 检查命令语法是否正确
2. 验证执行权限
3. 检查依赖环境
4. 查看错误输出信息

### 9. 如何分享脚本给其他用户？

**分享方法**:
1. 进入脚本编辑页面
2. 设置脚本权限
3. 选择可访问的用户/角色
4. 保存设置

### 10. 支持哪些脚本语言？

**支持列表**:
- Shell (Bash)
- Python
- Perl
- 其他可执行脚本

4. 查看系统日志
```

排查:

3. 测试脚本依赖
4. 查看执行日志
```

## 最佳实践

### 命令执行建议
- 执行前确认命令正确性
- 避免执行危险命令
- 使用脚本管理复杂操作
- 定期清理历史记录

### 脚本管理建议
- 为脚本添加详细说明
- 使用参数化提高复用性
- 定期更新和维护脚本
- 做好脚本版本管理
