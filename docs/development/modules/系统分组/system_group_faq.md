# 系统分组模块FAQ

## 常见问题

### 1. 一个资源可以属于多个分组吗？

**回答**: 当前版本一个资源只能属于一个分组。如果需要将资源移动到其他分组，需要先从原分组移除，再添加到新分组。

**未来规划**: 后续版本可能支持多分组关联。

### 2. 删除分组会影响分组内的资源吗？

**回答**: 不会。删除分组只会删除分组本身，分组内的资源不会被删除，只是解除了与该分组的关联关系。

### 3. 如何批量将资源添加到分组？

**操作步骤**:
1. 进入资源列表页面（如应用列表）
2. 启用多选模式
3. 选择需要分组的资源
4. 点击"添加到分组"按钮
5. 选择目标分组
6. 确认操作

### 4. 分组名称有什么限制？

**命名规则**:
- 长度：2-50个字符
- 支持中英文、数字、下划线、短横线
- 不允许重名
- 建议使用有意义的名称

### 5. 如何按分组查看资源？

**查看方法**:
1. 进入资源列表页面
2. 找到分组筛选器（通常在列表顶部）
3. 选择目标分组
4. 列表将只显示该分组下的资源

### 6. 分组支持哪些资源类型？

**支持类型**:
- 应用（App）
- 网站（Website）
- 数据库（Database）
- 容器（Container）
- 文件目录（File）

### 7. 如何修改分组信息？

**修改方法**:
1. 进入分组管理页面
2. 找到需要修改的分组
3. 点击"编辑"按钮
4. 修改分组名称或描述
5. 保存修改

**注意**: 分组类型创建后不可修改。

### 8. 分组可以嵌套吗？

**当前状态**: 暂不支持分组嵌套。

**替代方案**: 可以通过命名规范来体现层级关系，如"生产环境-Web服务"、"生产环境-数据库"。

### 9. 如何查看分组统计信息？

**查看方法**:
1. 进入分组管理页面
2. 每个分组卡片显示资源数量
3. 点击分组进入详情页查看详细统计

### 10. 分组数据会同步到其他设备吗？

**回答**: 分组数据存储在服务器端，只要使用同一账号登录，分组数据会在所有设备上同步显示。

## 故障排查指南

### 分组创建问题
```
错误: 分组名称已存在
解决:
1. 修改分组名称
2. 或删除已有同名分组
```

### 资源关联问题
```
错误: 资源已存在于其他分组
解决:
1. 先从原分组移除资源
2. 再添加到新分组
```

### 分组删除问题
```
错误: 分组删除失败
排查:
1. 检查分组是否存在
2. 确认操作权限
3. 查看系统日志
```

## 最佳实践

### 分组策略
- 按环境分组：开发、测试、生产
- 按业务分组：前端、后端、数据库
- 按项目分组：项目A、项目B
- 按团队分组：团队1、团队2

### 命名规范
- 使用统一的命名格式
- 包含环境或业务标识
- 便于识别和管理

### 管理建议
- 定期清理无用分组
- 保持分组结构清晰
- 为分组添加描述说明
- 指定分组负责人
